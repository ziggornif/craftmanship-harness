# Specs — agent briefs per slice

> One file per use-case slice, generated by `agent-brief` from the plan.
> Naming convention: `docs/specs/<NN>-<slice-name>.md`, numbered in build order.
> e.g. `docs/specs/01-register-user.md`, `docs/specs/02-place-order.md`
>
> This is the **only** committed per-slice artifact. The brainstorming design
> and the writing-plans plan are ephemeral working steps that fold into this
> brief — do **not** persist them under `docs/superpowers/` (CI fails if that
> tree appears; see the harness-layout sensor in `ci-setup`).
>
> See `skills/local/agent-brief/SKILL.md` for the brief format and review grid.
